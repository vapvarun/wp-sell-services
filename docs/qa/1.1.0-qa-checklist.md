# WP Sell Services 1.1.0 — QA Checklist

**Scope:** every feature shipped on 2026-04-21. Includes money-flow integrity
checks because most of these touch the wallet — assume any "Pass" requires
both UI behaviour AND DB-side money math to be correct.

**Assumed setup:**
- WP Sell Services + Pro both active on the 1.1.0 branch.
- Test users: 1 admin, 2 buyers (`buyer1`, `buyer2`), 2 vendors (`vendor1`,
  `vendor2`).
- Vendor1 has at least one published service.
- Commission rate set to 10% in admin settings.
- Email logging available (Mail-Catcher / WP Mail Log) so you can verify
  email dispatch.
- Use `?autologin=<slug>` (per `dev-auto-login.php`) to swap accounts fast.

Reset-between-tests:
- `DELETE FROM wp_options WHERE option_name LIKE '%wpss_email_cooldown_%';`
  (the 5-minute per-recipient/type rate-limit will silently swallow repeat
  test emails otherwise).

| 1.4 | Order detail view (vendor) — "Platform Fee" + "Your Earnings" rows | Both visible, sum to GROSS | `Platform Fee + Your Earnings == Total Amount` |
| 1.5 | Pro analytics page Revenue card | Renamed to "Net Earnings", uses NET values | Equal to wallet's Total Earned for the period |
| 1.6 | Pro analytics page Tips card | Separate card, shows tip-only NET | Equals wallet ledger SUM(amount) WHERE type='tip' |
| 1.7 | Pro wallet page Available Balance | Computed from `wp_wpss_wallet_transactions` (NOT user_meta `_wpss_wallet_balance`) | Run: `SELECT SUM(CASE WHEN type IN ('withdrawal','debit','dispute_refund') THEN -amount ELSE amount END) FROM wp_wpss_wallet_transactions WHERE user_id=X AND status='completed'` — must match the Available Balance card to the cent |
| 1.8 | Pro wallet Total Earned (3 orders) | Same source as Available Balance | — |

**Critical regression:** the old `_wpss_wallet_balance` user_meta is no
longer authoritative. Setting that meta directly (e.g. via
`update_user_meta`) must NOT change what the wallet UI shows.
