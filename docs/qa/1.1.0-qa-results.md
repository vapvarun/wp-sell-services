## Issues found + fixed during this run

### [blocker] Mobile dashboard nav labels
**Found:** at ≤1024px the sidebar collapsed to unlabelled icons — unusable on touch (no hover).
**Fixed in:** `b09ce14`.
**Approach:** grouped `[icon] Label` pills with visible group headings and 40px min tap-target.

### [blocker] Proposal AJAX ignored milestone contract fields
**Found:** `AjaxHandlers::submit_proposal` never read `$_POST['contract_type']` or `$_POST['milestones']` and hard-rejected on empty `price`. Milestone proposals silently failed with 200 OK and no DB row.
**Fixed in:** `3958a7e`.
**Approach:** handler now forwards both fields to `ProposalService::submit` and skips the price/days guards when `contract_type=milestone` (with a separate empty-phases guard).

### [blocker] Milestone accept redirect dead-end
**Found:** after accepting, buyer was sent to `/service-checkout?pay_order={$0 parent}` — which showed a bare "This order has already been paid." notice. No way forward.
**Fixed in:** `3958a7e`.
**Approach:** `AjaxHandlers::accept_proposal` detects milestone contracts and redirects to the dashboard order view instead of the service checkout.

### [blocker] Parent auto-complete silently no-oped
**Found:** the `wpss_milestone_approved` handler's `OrderService::update_status(parent, completed)` call returned false because `can_transition_naturally` didn't allow `in_progress → completed`.
**Fixed in:** `3958a7e`.
**Approach:** added `STATUS_COMPLETED` to the `in_progress` transition map. Milestone-contract parents skip `pending_approval` (each phase has its own submit+approve cycle), so the direct flip is the right move.

### [UX] Sub-order checkouts showed misleading 5-step stepper
**Found:** tip / extension / milestone pay-order pages showed "Pay → Requirements → Seller Works → Review → Complete" even though none of those steps apply to a one-shot credit or a phase-specific payment.
**Fixed in:** `82dff3c`.
**Approach:** `render_checkout_form` now detects sub-order platforms and renders a 2-step stepper with platform-specific copy (`Pay → Seller credited` / `Pay → Work continues on extended scope` / `Pay → Seller works on this phase`).

### [UX] Dead-end checkout notices with no back-link
**Found:** "This order has already been paid." and the locked-phase backstop were bare paragraphs. Buyers who bookmarked an old pay URL got stranded.
**Fixed in:** `82dff3c`.
**Approach:** both notices now include a "Back to order" secondary button that routes to the parent order view.
