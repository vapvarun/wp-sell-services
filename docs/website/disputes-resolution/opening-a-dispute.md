# Opening a Dispute

When issues with an order cannot be resolved through direct communication, you can open a formal dispute. This guide explains when and how to initiate the dispute process.

## What is a Dispute?

A dispute is a formal complaint about an order that requires marketplace intervention:

**Purpose**:
- Resolve conflicts between buyer and vendor
- Protect both parties' interests
- Facilitate fair resolution
- Document issues formally
- Enable admin mediation

**Not a Dispute**: Minor revision requests or communication issues that can be resolved directly with the vendor.

## When to Open a Dispute

### Valid Reasons for Disputes

Open a dispute when:

**Quality Issues**:
- Delivered work significantly differs from description
- Work quality is far below promised standard
- Deliverables are incomplete or missing elements
- Work is unusable or doesn't meet requirements

**Non-Delivery**:
- Vendor hasn't delivered after deadline passed
- Vendor stopped responding and hasn't delivered
- Vendor cannot or will not complete the order

**Scope Disputes**:
- Vendor demands extra payment for agreed work
- Vendor refuses to provide promised deliverables
- Service description was misleading

**Unauthorized Charges**:
- Charged for services not ordered
- Add-ons added without approval

**Intellectual Property**:
- Delivered work plagiarized or copied
- Work infringes copyright or trademark
- Stolen designs or content

**Breach of Terms**:
- Vendor violated marketplace policies
- Work delivered outside platform (against rules)

![When to dispute](../images/admin-dispute-reasons.png)

### When NOT to Dispute

Try resolving directly first for:

- Minor revision requests (use revision system)
- Communication delays under 48 hours
- Stylistic preferences or subjective opinions
- Reasonable delivery delays (if vendor communicates)
- Misunderstandings that can be clarified

**Best Practice**: Always attempt direct communication before opening a dispute.

## Before Opening a Dispute

### Step 1: Communicate with Vendor

Message the vendor first:

1. Clearly explain the issue
2. Provide specific examples
3. State what you need to resolve it
4. Give vendor 24-48 hours to respond
5. Document all communication

**Example Message**:
```
Hi [Vendor],

I've reviewed the delivered logo design but there are several
issues that need addressing:

1. The design uses red colors, but I specified blue in requirements
2. Missing the black & white version promised in Premium package
3. Files are JPG only - need vector files as described

Can you please provide the corrected deliverables within 48 hours?
I've attached the original requirements for reference.

Thanks,
[Your Name]
```

### Step 2: Use Revision Request

If work quality is the issue:

1. Submit a revision request through the order page
2. Be specific about what needs changing
3. Allow vendor to make corrections
4. Exhaust included revisions first

Only dispute if vendor refuses reasonable revisions.

### Step 3: Document Everything

Gather evidence before disputing:

- Screenshots of requirements
- Messages with vendor
- Delivered files
- Service description
- Any promises made
- Payment receipts

Strong documentation supports your dispute case.

## How to Open a Dispute

### Step-by-Step Process

1. **Navigate to Order Page**:
   - Go to **My Account → Orders**
   - Find the problematic order
   - Click to view order details

2. **Locate Dispute Button**:
   - Scroll to order actions
   - Click **Open Dispute** or **Report Problem**

![Open dispute button](../images/frontend-open-dispute-button.png)

3. **Select Dispute Reason**:
   - Choose primary issue from dropdown
   - Options typically include:
     - Quality Issues
     - Non-Delivery
     - Incomplete Delivery
     - Misleading Description
     - Vendor Unresponsive
     - Copyright/IP Issues
     - Other

4. **Provide Detailed Description**:
   - Explain the issue thoroughly
   - Include specific examples
   - Reference requirements or promises
   - State what resolution you seek
   - Keep tone professional and factual

![Dispute form](../images/frontend-dispute-form.png)

**Good Dispute Description**:
```
I ordered a WordPress website redesign (Premium package, $500).
The vendor delivered the work 3 days late and it has multiple
issues:

1. Design doesn't match the mockup I approved
2. Site is not responsive on mobile devices (requirement #3)
3. Contact form doesn't work
4. Missing blog section promised in Premium package

I've requested revisions twice over the past week. Vendor made
minor changes but hasn't addressed mobile responsiveness or
missing blog. Vendor stopped responding 5 days ago.

I'm seeking either:
- Complete revision to match requirements, OR
- 50% refund due to incomplete deliverables

Attached: Original requirements, approved mockup, screenshots
of issues, message history.
```

5. **Upload Evidence**:
   - Attach supporting files
   - Include screenshots
   - Upload correspondence
   - Provide before/after comparisons
   - Maximum file size: 5-10MB per file

6. **Select Desired Outcome**:
   - Full refund
   - Partial refund
   - Order completion
   - Revision/correction
   - Compensation

7. **Review and Submit**:
   - Review all information
   - Ensure accuracy
   - Click **Submit Dispute**
   - Receive confirmation

### Dispute Submission Confirmation

After submitting:

- Dispute ID assigned (e.g., #DISP-12345)
- Email confirmation sent
- Order status changes to "Disputed"
- Vendor notified of dispute
- Admin review begins
- You can view dispute in dashboard

![Dispute confirmation](../images/frontend-dispute-confirmation.png)

## What Happens to Your Order

### Order Status Change

When dispute opens:

- **Order Status**: Changes to "Disputed"
- **Order Timer**: Paused (no late delivery penalties)
- **Payment Hold**: Funds held in escrow
- **Work Delivery**: Vendor can still upload deliverables
- **Revisions**: Revision system may be locked
- **Regular Actions**: Some order actions disabled

### Payment Protection

Your payment is protected during disputes:

- Funds held by marketplace (not released to vendor)
- Cannot be withdrawn by vendor
- Remains in escrow until resolution
- Refunded if dispute decided in your favor

## Dispute Timeline

### Typical Timeline

| Stage | Duration | Description |
|-------|----------|-------------|
| **Submitted** | Immediate | Dispute opened, parties notified |
| **Initial Response** | 2-3 days | Vendor submits response |
| **Evidence Exchange** | 3-7 days | Both parties provide evidence |
| **Admin Review** | 3-5 days | Marketplace admin investigates |
| **Resolution** | 1-2 days | Decision made and applied |
| **Appeal Period** | 3-7 days | Time to appeal decision (optional) |

**Total Time**: Most disputes resolve in 7-14 days.

### Response Requirements

**For Buyers**:
- Respond to admin questions within 48 hours
- Provide requested evidence promptly
- Check dispute status daily

**For Vendors**:
- Vendor has 48-72 hours to respond
- Must provide their side of the story
- Can upload corrective deliverables

Failure to respond may result in automatic decision against non-responsive party.

## Communication During Disputes

### Dispute Messages

Dedicated communication channel within dispute:

- **Separate from Order Messages**: Dispute has its own thread
- **Admin Visibility**: All messages visible to admin
- **Both Parties**: You and vendor can message
- **Evidence**: Attach files to messages
- **Professional Tone**: Keep communication respectful

![Dispute message thread](../images/admin-dispute-messages.png)

### Message Guidelines

**Do**:
- Stick to facts
- Provide additional evidence
- Answer admin questions directly
- Remain professional
- Focus on resolution

**Don't**:
- Make personal attacks
- Use threatening language
- Spam multiple messages
- Share private information
- Communicate outside platform about dispute

## Who Can Open a Dispute

### Buyer-Initiated Disputes

Buyers can dispute when:

- Order has been paid
- Issue occurred with the order
- Delivery was made or deadline passed
- Direct resolution attempts failed

### Vendor-Initiated Disputes

Vendors can also open disputes for:

- Buyer providing false information
- Buyer requesting work outside original scope
- Buyer refusing to pay for completed work
- Buyer making unreasonable demands
- Violation of platform terms by buyer

### Timeframe for Disputes

Disputes can be opened:

- **During Order**: While order is active
- **After Delivery**: Within 14-30 days of completion (varies)
- **Not After**: Typically cannot dispute after 30 days of completion

Check your marketplace's dispute window policy.

## Dispute Limitations

### One Dispute Per Order

- Only one formal dispute per order
- If closed, generally cannot reopen
- Must include all issues in initial dispute
- Add new evidence to existing dispute if needed

### Frivolous Disputes

Avoid opening disputes for:

- Buyer's remorse (changed mind)
- Subjective preferences
- Unreasonable expectations
- Minor, fixable issues
- Attempting to scam free work

**Consequences**: Abuse of dispute system may result in buyer account penalties.

## Impact of Opening a Dispute

### On Your Order

- Order progress pauses
- Delivery deadlines freeze
- Cannot leave review until resolved
- Order shows "Disputed" status publicly

### On Vendor

- Vendor's metrics affected
- Active dispute count visible
- May impact verification level
- Funds held in escrow
- Resolution affects rating

### On Your Relationship

- Direct trust damaged
- Future working relationship unlikely
- Other vendors may see dispute history
- Platform tracks dispute patterns

**Best Case**: Mutual resolution during dispute process can restore relationship.

## Tips for Successful Disputes

### Present Your Case Clearly

1. **Chronological Order**: Explain what happened step by step
2. **Specific Issues**: List concrete, verifiable problems
3. **Evidence**: Attach proof for every claim
4. **Requirements Reference**: Show what was promised vs delivered
5. **Communication Record**: Demonstrate attempt to resolve directly

### What Admins Look For

Admins assess disputes based on:

- **Evidence Quality**: Documentation and proof
- **Service Description**: What was promised
- **Order Requirements**: What buyer requested
- **Deliverables**: What vendor provided
- **Communication**: How both parties interacted
- **Platform Policies**: Compliance with terms
- **Industry Standards**: Reasonable expectations

### Be Reasonable

- Request fair resolution (don't overreach)
- Acknowledge any partial completion
- Consider partial refund if some work is usable
- Focus on facts, not emotions
- Accept that subjective issues are hard to judge

## What You Should NOT Do

### Avoid These Mistakes

1. **Don't Threaten**: "I'll leave bad reviews everywhere"
2. **Don't Blackmail**: "Refund me or I'll report you"
3. **Don't Lie**: False claims will backfire
4. **Don't Spam**: One dispute is enough
5. **Don't Go Off-Platform**: Keep everything documented on marketplace
6. **Don't Accept Outside Deals**: Vendor offering refund via PayPal separately
7. **Don't Abuse System**: Using disputes to get free work

These actions can result in dispute being decided against you or account suspension.

## After Opening a Dispute

### Monitor Dispute Status

Check regularly at **My Account → Disputes**:

- View dispute status
- Read new messages
- Upload additional evidence
- Respond to admin requests
- Track progress

### Respond to Admin Requests

Admins may ask for:

- Additional evidence
- Clarification of issues
- Specific documentation
- Your preferred resolution
- Response to vendor's claims

**Respond within 48 hours** to avoid delays.

### Continue Professional Communication

Even during disputes:

- Remain respectful to vendor
- Focus on facts, not personal attacks
- Answer questions directly
- Provide requested information
- Work toward resolution

Learn more about the [dispute resolution process](dispute-process.md).

## Dispute Alternatives

### Cancellation Request

For early-stage orders:

- Request order cancellation instead
- Both parties agree to cancel
- Refund processed without dispute
- Faster than formal dispute

### Marketplace Mediation

Some platforms offer informal mediation:

- Contact support before formal dispute
- Support team facilitates conversation
- May resolve without formal dispute process

### Negotiated Resolution

Work directly with vendor:

- Partial refund agreement
- Extended delivery time
- Reduced scope compromise
- Document any agreement
