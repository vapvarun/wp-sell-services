## Final wrap steps (after Wave 2)

1. **Pull the branch** locally to ensure all agent commits landed:
   - `cd wp-sell-services && git pull origin 1.1.0`
   - `cd wp-sell-services-pro && git pull origin 1.1.0`
2. **Run the QA checklist** at `docs/qa/1.1.0-qa-checklist.md`:
   - Every row in sections 1-15 must pass or have a documented reason.
   - Section 13 (DB integrity queries) is the money-math proof.
3. **Spot-check the 3 audit docs** → every Critical + High finding is closed.
4. **Run PHPCS + PHPStan** on both plugins:
   - `cd wp-sell-services && composer phpcs`
   - `cd wp-sell-services-pro && composer phpcs`
5. **Regenerate translation POT** if Packet A added new strings:
   - WP CLI: `wp i18n make-pot wp-sell-services languages/wp-sell-services.pot`
6. **Bump plugin header** in `wp-sell-services.php` and `wp-sell-services-pro.php`:
   - Version: 1.1.0
   - Stable tag in `readme.txt`: 1.1.0
7. **Tag the releases:**
   - `git tag -a v1.1.0 -m "Release 1.1.0" && git push origin v1.1.0` in both plugins.
8. **Build release zips** if applicable — follow `wp-releaser` workflow.
9. **Publish docs to live** once packet G confirms the LOCAL preview:
   - `mcp__wbcom-docs__publish_product_docs({ ..., sync_to_live: true })`

## Coordination rules

- Wave 1 agents are partitioned so they don't share files. If two agents
  commit to the same file (e.g. both touch a CSS file), reconcile by
  serialising them rather than resolving merge conflicts manually.
- Any agent that discovers a bug outside its scope: add a finding to
  `docs/qa/1.1.0-dataflow-audit.md` (or the relevant audit doc) rather
  than fix it, so the fix can be triaged into its own packet.
- All commits to `1.1.0` only. Do not merge to `main` until every Wave
  completes and QA passes.

## Risks + mitigations for the final wrap

- **Composer autoload drift** — each packet that adds a new class must
  run `composer dump-autoload -q`. Packet G is the highest risk because
  it adds no classes, but Packet B might. Verify before tagging.
- **Mobile screenshots may pile up in `docs/qa/screenshots/`** — this
  directory may or may not be `.gitignore`d. Check before committing
  binary blobs in Packet C's output.
- **POT regen may show the language audit changes + packet A + dozens
  of new strings from Wave 1** — keep the POT commit separate so a
  translator can diff it easily.
- **Pro plugin double-commit risk** — Packet F writes to Pro; no other
  packet does, so no conflict there.
