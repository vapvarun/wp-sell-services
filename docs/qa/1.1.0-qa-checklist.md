## Setup before you start

**Required active plugins:** WP Sell Services + WP Sell Services Pro (both on the `1.1.0` branch).

**Required test users:**

| Role | Username | Notes |
|---|---|---|
| Admin | `admin` | Full access |
| Buyer 1 | `buyer1` | Plain customer |
| Buyer 2 | `buyer2` | Second customer for cross-account checks |
| Vendor 1 | `vendor1` | At least 1 published service |
| Vendor 2 | `vendor2` | Second vendor for cross-account checks |

Use `?autologin=<username>` to swap accounts (the `dev-auto-login.php` mu-plugin is already in place).

**Required settings:**

- Commission rate: **10%** (Settings → Commission)
- Email logging: **Mail-Catcher** or **WP Mail Log** plugin active so you can verify dispatch
- Pro license: **active** (Settings → License)

**Reset between test sessions:**

```sql
DELETE FROM wp_options WHERE option_name LIKE '%wpss_email_cooldown_%';
```

(The 5-minute per-recipient/type cooldown will silently swallow repeat test emails otherwise.)

**How to mark items:** put `[x]` in the checkbox when an item passes; if it fails, leave `[ ]` and add a `**FAIL:** <description>` line directly under the row. File the failure as a Basecamp bug card with the section + row number (e.g. "QA §7d.7 — pay-order URL tampering not blocked").
