# Project Milestones

Break large orders into smaller deliverable phases with individual approval steps and payments. Perfect for complex projects that take weeks or months to complete.

## What are Milestones?

Milestones allow vendors and buyers to divide big projects into manageable chunks called "milestones." Each milestone has its own:
- Description and deliverables
- Deadline
- Payment amount (percentage of total order)
- Approval process

Instead of waiting until the entire project is done, work is reviewed and paid for in stages.

**Example: Website Design Project**
```
Total Order: $2,000 (30 days)

Milestone 1: Wireframes & Mockups
- Due: Day 7
- Payment: $500 (25%)

Milestone 2: Homepage Development
- Due: Day 14
- Payment: $600 (30%)

Milestone 3: Inner Pages
- Due: Day 21
- Payment: $600 (30%)

Milestone 4: Final Revisions & Launch
- Due: Day 30
- Payment: $300 (15%)
```

## Why Use Milestones?

### For Buyers

**Benefits:**
- ✅ See progress throughout the project
- ✅ Review work in stages (catch issues early)
- ✅ Pay as work is completed (not all upfront)
- ✅ Clear expectations at each phase
- ✅ Easier to provide feedback
- ✅ Less risk on large projects

**When to Request Milestones:**
- Projects over $500
- Projects taking more than 2 weeks
- Complex deliverables
- Multiple phases or components
- First time working with vendor
- Custom development work

### For Vendors

**Benefits:**
- ✅ Get paid throughout the project (better cash flow)
- ✅ Clear scope for each phase
- ✅ Buyer stays engaged
- ✅ Reduce scope creep
- ✅ Show progress to keep buyers confident
- ✅ Easier project management

**When to Offer Milestones:**
- Large custom projects
- Long-term engagements
- Projects with multiple deliverable types
- High-value orders
- Work requiring buyer input at stages

## How Milestones Work

### Milestone Workflow

```
1. Vendor creates milestones → 2. Buyer reviews/approves plan
                                        ↓
6. Payment released to vendor ← 5. Buyer approves milestone
                                        ↑
                               4. Vendor submits deliverable
                                        ↑
                               3. Vendor works on milestone
```

### Milestone Statuses

| Status | What It Means | Who Acts Next |
|--------|---------------|---------------|
| **Pending** | Milestone created, not started | Vendor (start work) |
| **In Progress** | Vendor actively working | Vendor (submit deliverable) |
| **Submitted** | Deliverable uploaded for review | Buyer (approve/reject) |
| **Approved** | Buyer accepted the work | Next milestone or completion |
| **Rejected** | Needs changes/revisions | Vendor (revise and resubmit) |

### Payment Release

Payments are released **per milestone** when approved:

**Traditional Order:**
```
Order Total: $1,000
Vendor Works for 30 Days
Vendor Delivers
Buyer Approves
Vendor Receives: $1,000 (on day 30)
```

**With Milestones:**
```
Milestone 1 Approved (Day 7): $250 released
Milestone 2 Approved (Day 14): $300 released
Milestone 3 Approved (Day 21): $300 released
Milestone 4 Approved (Day 28): $150 released

Vendor receives payment throughout project
```

## Setting Up Milestones (Vendors)

### Creating Milestones

Milestones can be created:
- **Before order acceptance:** Add to order proposal
- **After order acceptance:** Propose to buyer for approval
- **Buyer requests:** Buyer asks vendor to create milestones

**Access:** Order Details → **Milestones** tab → **Add Milestone**

### Step-by-Step Setup

1. **Go to Order Details:**
   - Navigate to **Vendor Dashboard → Orders**
   - Open the order
   - Click **Milestones** tab

2. **Click "Add Milestone":**
   - Button at top of milestones section
   - Opens milestone creation form

3. **Fill Milestone Details:**

   **Milestone Title:**
   - Clear, descriptive name
   - Example: "Homepage Design Mockup"

   **Description:**
   - What will be delivered
   - Specific requirements
   - Format/file types
   - Example: "3 homepage design concepts in PSD format with desktop and mobile versions"

   **Payment Amount:**
   - Dollar amount or percentage
   - Must total 100% of order value
   - Example: $400 or 25%

   **Deadline:**
   - Select due date
   - Must be before final order deadline
   - Realistic timeframe

   **Deliverables:**
   - List specific files/outputs
   - Example: "homepage-mockup-v1.psd, homepage-mockup-v2.psd, homepage-mockup-v3.psd"

4. **Save Milestone:**
   - Click **Create Milestone**
   - Milestone added to order
   - Buyer notified

5. **Repeat for All Phases:**
   - Add remaining milestones
   - Ensure total payment = 100%
   - Check deadlines are sequential

![Add milestone form](../images/admin-add-milestone.png)

### Milestone Planning Best Practices

**Break Down Logically:**
```
Good Example (Logo Design):
✅ Milestone 1: Research & Concepts (3 concepts)
✅ Milestone 2: Refinement (2 revised concepts)
✅ Milestone 3: Final Design (all formats)

Bad Example:
❌ Milestone 1: Some work
❌ Milestone 2: More work
❌ Milestone 3: Final stuff
```

**Payment Distribution:**
- Front-load slightly (covers setup time)
- Example: 30%, 30%, 25%, 15%
- Don't make final milestone too small (buyer may delay approval)

**Realistic Deadlines:**
- Allow time for buyer feedback
- Account for revisions
- Build in buffer days
- Consider dependencies

**Clear Deliverables:**
- Be specific about what's included
- List file formats
- Mention quantity (e.g., "5 pages")
- Note what's NOT included

### Editing Milestones

**Before Work Starts:**
- Click **Edit** on milestone
- Modify any field
- Save changes
- Buyer notified of changes

**After Work Starts:**
- Can only edit description and deadline
- Cannot change payment amount
- Requires buyer approval for major changes

**Reordering Milestones:**
- Drag and drop to reorder
- Updates sequence automatically
- Deadlines should remain sequential

## Working on Milestones (Vendors)

### Starting a Milestone

1. **Change Status to "In Progress":**
   - Order Details → Milestones
   - Click milestone
   - Click **Start Milestone**

2. **Track Your Work:**
   - Monitor days remaining
   - Check deliverables checklist
   - Communicate progress with buyer

3. **Send Progress Updates:**
   - Message buyer with updates
   - Share work-in-progress screenshots
   - Ask clarifying questions early

### Submitting a Milestone

When work is complete:

1. **Prepare Deliverables:**
   - Organize all files
   - Name files clearly
   - Test everything works
   - Create README if needed

2. **Submit Milestone:**
   - Click **Submit Milestone** button
   - Upload deliverable files (same as regular delivery)
   - Write submission message
   - Explain what's included

3. **Submission Message Example:**
   ```
   Hi [Buyer Name],

   I've completed Milestone 2: Homepage Development.

   Included Files:
   - homepage-final.html (main homepage file)
   - styles.css (all styling)
   - script.js (interactive elements)
   - images.zip (all optimized images)

   The homepage is fully responsive and tested on:
   - Chrome, Firefox, Safari
   - Desktop, tablet, mobile

   Please review and let me know if any adjustments are needed.

   Thanks!
   ```

4. **Wait for Approval:**
   - Milestone status → **Submitted**
   - Buyer reviews deliverable
   - Payment held until approval

![Submit milestone](../images/admin-submit-milestone.png)

### If Milestone is Rejected

When buyer requests changes:

1. **Review Feedback:**
   - Read buyer's rejection message
   - Understand what needs changing
   - Ask questions if unclear

2. **Make Revisions:**
   - Update deliverables
   - Address all feedback points
   - Test changes

3. **Resubmit:**
   - Upload revised files
   - Explain changes made
   - Reference buyer's feedback

4. **Rejection vs. Revision:**
   - Rejected milestone: Major changes needed
   - Revision request: Minor tweaks
   - Communicate to clarify expectations

**Note:** Milestone rejections don't count against order revisions. Each milestone can be revised independently.

## Reviewing Milestones (Buyers)

### When Vendor Submits

You receive notification:
- Email alert
- Dashboard notification
- Order page shows "Milestone Ready for Review"

### Review Process

1. **Access Milestone:**
   - Go to **My Orders → [Order Name]**
   - Click **Milestones** tab
   - Find submitted milestone

2. **Download Deliverables:**
   - Click **Download Files**
   - Extract and review all files
   - Test functionality (if applicable)
   - Compare to milestone description

3. **Decision Time:**

   **Option A: Approve Milestone**
   - Click **Approve Milestone**
   - Confirm approval
   - Payment released to vendor
   - Next milestone begins

   **Option B: Request Revisions**
   - Click **Request Changes**
   - Describe changes needed:
     - Be specific
     - Reference milestone requirements
     - Attach markup/screenshots if helpful
   - Submit revision request
   - Vendor makes changes and resubmits

![Review milestone](../images/frontend-review-milestone.png)

### Approval Checklist

Before approving, verify:

- ✅ All deliverables included?
- ✅ Matches milestone description?
- ✅ Quality meets expectations?
- ✅ Files work/open correctly?
- ✅ Any issues need addressing?

### Revision Request Guidelines

**Be Clear and Specific:**
```
❌ Bad: "This doesn't look good"
✅ Good: "Please change the header background to navy blue (#003366)
         and increase the logo size by 20%"

❌ Bad: "Not what I wanted"
✅ Good: "The layout should have the image on the left, not right,
         as shown in the reference I provided"
```

**Reasonable Requests:**
- Minor design tweaks
- Bug fixes
- Alignment with original requirements
- Addressing missed specifications

**Out of Scope:**
- Complete redesign
- New features not in milestone description
- Additional deliverables
- Changed requirements from original order

### Auto-Approval

If you don't review a milestone:
- Reminder sent after 3 days
- Second reminder after 5 days
- Auto-approved after 7 days (configurable)
- Payment automatically released

**Set calendar reminders** to review milestones promptly!

## Payments and Milestones

### Payment Schedule

**How Money Flows:**

1. **Buyer Pays Order Total Upfront:**
   - Full order amount charged at checkout
   - Example: $2,000 charged to buyer's card

2. **Funds Held in Escrow:**
   - Payment held by marketplace
   - Released per milestone approval
   - Not accessible to vendor yet

3. **Per-Milestone Release:**
   ```
   Milestone 1 Approved → $500 released to vendor
   Milestone 2 Approved → $600 released to vendor
   Milestone 3 Approved → $600 released to vendor
   Milestone 4 Approved → $300 released to vendor

   Total Released: $2,000
   ```

4. **Vendor Withdraws:**
   - Funds added to vendor wallet as each milestone is approved
   - Can withdraw after meeting minimum withdrawal amount

![Payment flow diagram](../images/admin-milestone-payment-flow.png)

### Commission Deduction

**Commission deducted from each milestone payment:**

```
Milestone Payment: $500
Commission (10%): $50
Vendor Receives: $450

Milestone Payment: $600
Commission (10%): $60
Vendor Receives: $540
```

**Alternative: Commission on Final Milestone Only** (if configured by admin)

```
Milestones 1-3: No commission deducted
Milestone 4: All commission deducted at once
```

Admin chooses commission timing in settings.

### Partial Refunds

If order cancelled mid-project:

**Approved Milestones:**
- Payment already released
- Vendor keeps this amount
- No refund to buyer

**Unapproved Milestones:**
- Payment returned to buyer
- Not released to vendor

**Example:**
```
4 Milestones × $500 each = $2,000 total

Milestones 1 & 2 Approved: $1,000 released to vendor
Project Cancelled During Milestone 3

Refund to Buyer: $1,000 (Milestones 3 & 4)
Vendor Keeps: $1,000 (Milestones 1 & 2)
```

- Change milestone sequence

### Milestone Settings

Configure milestones: **Settings → Orders → Milestones**

| Setting | Options | Default |
|---------|---------|---------|
| Enable Milestones | Yes/No | Yes |
| Who Can Create | Vendors Only / Both / Buyer Request | Vendors Only |
| Require Buyer Approval | Yes/No | Yes |
| Auto-Approve Milestones | Days (0 = disabled) | 7 days |
| Minimum Order Value | Amount to enable milestones | $500 |
| Max Milestones per Order | Number | 10 |
| Commission Per Milestone | Yes/No (or final only) | Yes |

## Best Practices

### For Vendors

**Planning Milestones:**
- ✅ Create milestones BEFORE starting work
- ✅ Get buyer agreement on milestone plan
- ✅ Make each milestone independently valuable
- ✅ Front-load milestones (30-35% in first milestone)
- ✅ Build in time for buyer review

**Milestone Size:**
- 3-5 milestones for most projects
- Too many = administrative overhead
- Too few = loses milestone benefits

**Communication:**
- Update buyer on progress
- Send work-in-progress previews
- Ask questions early
- Confirm deliverables before submitting

### For Buyers

**Reviewing Milestones:**
- ✅ Review within 2-3 days
- ✅ Test all deliverables thoroughly
- ✅ Provide specific feedback
- ✅ Approve promptly if satisfied
- ✅ Don't approve to "be nice" if issues exist

**Collaboration:**
- Respond to vendor questions quickly
- Provide feedback at each stage
- Don't wait to share concerns
- Stay engaged throughout project

### For Admins

**Encourage Milestone Use:**
- Set minimum order value ($500+)
- Provide milestone templates
- Educate vendors on benefits
- Highlight in onboarding

**Monitor Issues:**
- Watch for stuck milestones
- Check approval times
- Identify problematic patterns
- Step in when needed

## Frequently Asked Questions

### Can milestones be added after the order starts?

Yes, but both parties must agree. Vendor proposes milestones, buyer approves. Best practice is to set them up before work begins.

### What if we can't agree on milestone plan?

Contact admin for mediation. Admin can help structure fair milestones or decide if project should proceed without them.

### Can milestone deadlines be extended?

Yes. Vendor can request extension on individual milestones (separate from overall order deadline). Buyer approves or denies.

### What happens if a milestone is late?

Order isn't marked "late" until the final deadline. Individual milestone delays are between vendor and buyer. Late milestones may delay subsequent ones.

### Can milestone payments be changed after creation?

No, once buyer approves the milestone plan. Protects both parties. Admin can modify if both parties agree.

### What if the buyer never approves a milestone?

Auto-approval after 7 days (default). Protects vendors from unresponsive buyers. Buyer receives reminders before auto-approval.

### Do milestones affect revisions?

No. Order-level revisions are separate from milestone revisions. Each milestone can be revised independently without using order revisions.

### Can I skip milestones and just deliver everything at once?

Technically yes, but defeats the purpose. If you submit all milestones simultaneously, it's essentially a regular order.

### What if project scope changes mid-milestone?

Discuss with buyer. May require:
- Modifying milestone descriptions
- Adding new milestones
- Creating new order for additional work
- Admin mediation if disagreement

### Are milestones required for large orders?

Not required, but recommended for orders over $500 or 14+ days. Admin can set threshold to require milestones automatically.

## Troubleshooting

### Can't Create Milestones

**Possible Causes:**
- Order already started
- Order value below minimum threshold
- Milestones disabled by admin
- Already at maximum milestones

**Solution:** Check order status and settings. Contact admin if milestones should be available.

### Milestone Percentages Don't Add to 100%

When creating milestones, ensure:
- Sum of all milestone payments = order total
- System may show warning if mismatch
- Cannot save milestone plan until totals match

**Fix:** Adjust milestone amounts until total = 100%.

### Buyer Won't Approve Milestone

**Steps:**
1. Message buyer to check milestone status
2. Ask for specific feedback
3. Wait for auto-approval (if enabled)
4. Contact admin if buyer truly unresponsive

**Admin Can:**
- Manually approve milestone
- Mediate disagreement
- Adjust deadline

### Payment Not Released After Approval

**Check:**
- Milestone status shows "Approved"?
- Commission deducted correctly?
- Check wallet transaction history
- May take few minutes to process

**Still Missing:** Contact admin with order and milestone details.

### Can't Edit Milestone After Creation

**Restrictions:**
- Can't change payment amount after buyer approves plan
- Can't edit after submission
- Can only edit pending/in-progress milestones

**Need Major Changes?** Discuss with buyer and admin. May need to cancel and recreate milestones.
