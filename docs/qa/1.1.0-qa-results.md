## Issue fixed during this run

### [fix] Mobile dashboard nav labels
**Found:** at ≤1024px the dashboard sidebar collapsed to an unlabelled icon strip. Three `display: none` rules hid group headings ("BUYING / SELLING / ACCOUNT") and every nav-item label text. Users on phones had to guess what each icon meant and touch gives no hover tooltip.

**Fixed in:** commit `b09ce14` on branch `1.1.0`.

**Approach:** at mobile, nav renders as grouped pill lists — section heading above, then `[icon] Label` pills wrapping across rows. Tap-targets ≥40px, active pill uses `--wpss-primary`. Mirror change applied to RTL bundle.
