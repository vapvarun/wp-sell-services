## What shipped in 1.1.0

### Features

- **Milestone contracts** (Upwork-style) — vendor picks contract type at proposal time; milestones pre-created on acceptance; lock-step payment; auto-complete parent on final approval; cascade-cancel unpaid phases.
- **Paid extensions** — catalog-order add-ons; quote → pay → parent-deadline push; mutual exclusion with milestones.
- **Tipping** — commission split at payment; dedicated email + receipt view.
- **Vendor intro video** — YouTube/Vimeo on profile + service detail page.
- **NET earnings ledger + CSV export** — period selector, filtered export, accounting-ready summary block.
- **Sub-order pattern** — shared architecture across tips / extensions / milestones; `platform` marker + `wpss_order_paid` credit handler + abandon-cron contract with contract-milestone carve-out.

### Quality / infrastructure

- **Lucide icons** everywhere (frontend + backend). 0 Dashicons in templates/src (except 2 Lucide-shaped base64 SVGs in `add_menu_page` calls — WP API constraint).
- **CSS tokens** — 1,135 hardcoded hexes refactored to `var(--wpss-TOKEN, #fallback)` across 30 CSS files. Theme white-label via `:root` override proven.
- **Mobile responsive** — every 1.1.0 component verified at 390px / 640px / 1024px.
- **Language** — "Phase" as user noun, "Milestone" as feature name. Decision doc at `docs/decisions/milestone-terminology.md`. 0 bare English strings in owned files.
- **REST parity** — milestone / extension / proposal endpoints at `wpss/v1/` with lock-step 409, ownership checks, standardised error envelope. 58-assertion PHPUnit REST test passes.
- **Money-flow integrity** — tip idempotency (sub-order keyed), deferred-hook transaction in convert_to_order, per-sub-order rate-limit scope, mark_as_paid sub-order skip, Pro double-credit guard.

## Deferred to 1.1.1

These items were captured but intentionally not shipped in 1.1.0:

- **H-2 structural compliance** (per `~/.claude/skills/wp-plugin-development/` audit):
  - `{prefix}_rest_prepare_{resource}` response filters on every REST endpoint
  - `before_*` filterable hooks (returnable `WP_Error`) alongside `after_*` actions on every write
  - `has_more = (offset + count) < total` audit on list endpoints with COUNT query pairs
  - `WP_Error` return types instead of `{success, message}` array pattern
  - Tap-target ≥ 40×40px audit
  - Specificity-match audit on mobile overrides
  - Escaping rule sweep (`esc_html( number_format_i18n() )`, `wp_kses_post( paginate_links() )`)
  - Replace browser `confirm()` / `alert()` calls with toast/modal utility
  - ARIA + focus-trap audit on modals
- **Email templates — inline Lucide SVG strategy** (email clients don't run JS so `data-lucide` doesn't hydrate — 7 email templates still use raw `<svg>` from before the migration)
- **Per-milestone disputes** (v1 = parent-level)
- **Auto-approve timer** on submitted milestones
- **Attachments on milestone delivery** (schema ready, UI text-only for v1)
- **Actor-based cancel/decline labels** ("Declined by you" vs "Declined by buyer") — needs new DB meta column
