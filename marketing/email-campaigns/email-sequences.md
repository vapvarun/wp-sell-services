### Email 3: Order Management (Week 3)

**Subject:** How the complete order workflow handles everything from payment to delivery

**Preview Text:** 11 order statuses track every stage. Requirements, messaging, deliveries, revisions, disputes - all built in.

```markdown
Hi [Name],

Here's what happens when someone orders a service on your marketplace.

**The complete order lifecycle:**

1. **Pending Payment** - Buyer checks out through WooCommerce
2. **Pending Requirements** - Buyer submits project details
3. **In Progress** - Vendor works on the order
4. **Pending Approval** - Vendor submits delivery
5. **Revision Requested** (if needed) - Buyer requests changes
6. **Completed** - Order finished and approved

Plus 5 additional statuses for edge cases: Late, Cancelled, Disputed, On Hold, and Pending Review.

**Built-in messaging per order:**

Buyers and vendors communicate directly within each order. File attachments supported. All messages stored with the order.

**Delivery and revision system:**

Vendors upload final files when work is complete. Buyers can request revisions if the package includes them (configurable per package). If the buyer doesn't respond within 3 days (configurable), the order auto-completes.

**Deadline tracking:**

Delivery deadlines are calculated automatically from package delivery time plus any addon modifiers. A cron job checks hourly for late orders. Vendors get a reminder 24 hours before deadline.

**Extension requests:**

If a vendor needs more time, they can request a deadline extension. Buyers approve or deny the request.

**Disputes:**

If something goes wrong, buyers or vendors can open a formal dispute. Both parties submit evidence, and you (the admin) mediate and enforce the resolution: full refund, partial refund, additional revision, or mark as resolved.

All of this runs automatically. You set the rules in settings, and the plugin handles the workflow.

    <div style="text-align: center; margin-bottom: 30px;">

    </div>

    <div style="background: #fff; padding: 30px; border-radius: 8px;">
        <!-- Email content here -->

        <div style="margin: 30px 0; text-align: center;">
            <a href="[CTA_URL]" style="display: inline-block; background: #0073aa; color: #fff; padding: 12px 30px; text-decoration: none; border-radius: 4px; font-weight: 600;">[CTA TEXT]</a>

        <p><a href="[UNSUBSCRIBE_URL]" style="color: #0073aa;">Unsubscribe</a></p>
    </div>

</body>
</html>
```

### Typography Guidelines

- **Headings:** Bold, 18-20px
- **Body text:** 16px, line-height 1.6
- **Lists:** Use bullet points (•) or checkmarks (✓)
- **Links:** Blue (#0073aa), underlined on hover
- **CTA buttons:** High contrast, 12-14px padding, rounded corners

### Personalization Tokens

Replace these tokens with your email platform's merge tags:

- `[Name]` - Recipient first name
- `[Email]` - Recipient email
- `[Site_URL]` - Their WordPress site URL
- `[Days_Since_Install]` - Days since they installed the plugin
- `[PRICE]` - Pro version pricing
- `[Company_Name]` - Your company name
- `[CTA_URL]` - Call-to-action link
- `[UNSUBSCRIBE_URL]` - Unsubscribe link

### A/B Testing Recommendations

**Subject Lines:**

Test direct value ("Launch your marketplace") vs. curiosity ("Your WordPress site just became...") vs. feature-focused ("Complete order workflow included").

**CTA Placement:**

Test single CTA at bottom vs. CTA at top and bottom vs. multiple CTAs throughout.

**Length:**

Test short emails (under 150 words) vs. detailed feature explanations (400+ words).

**Proof Points:**

Test emails with customer testimonials vs. feature lists vs. use case scenarios.

## Success Metrics to Track

### Email Performance

- **Open rate target:** 25-35%
- **Click-through rate target:** 3-8%
- **Conversion rate target:** 1-3% (download to install)
- **Unsubscribe rate acceptable:** Under 0.5%

### Campaign Performance

- **Downloads from email:** Track via UTM parameters
- **Pro upgrades from email:** Track via coupon codes or UTM
- **Support questions generated:** Track via reply-to monitoring
- **Demo requests:** Track via dedicated demo link in emails

### Optimization Points

If open rates are low, test subject lines and preview text
If CTR is low, test CTA placement and button text
If conversion is low, review landing page alignment with email messaging
If unsubscribes are high, reduce email frequency or improve targeting

## Legal Compliance Notes

### Required Elements

- Physical mailing address in footer
- Unsubscribe link in every email
- "You're receiving this because..." explanation
- Company name and contact information

### GDPR Compliance

- Only send to users who opted in
- Honor unsubscribe requests within 48 hours
- Include data processing information if required
- Allow users to request data deletion

### CAN-SPAM Compliance

- Accurate "From" name and email address
- Relevant subject line (no deceptive claims)
- Clear identification as advertisement (if applicable)
- Physical postal address
- Unsubscribe mechanism that works for 30+ days
